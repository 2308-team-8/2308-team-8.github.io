A zero-downtime, reversible database schema migration tool for PostgreSQL

Case Study

1) Introduction

Laridae is a zero-downtime, reversible database schema migration tool for PostgreSQL. Laridae utilizes GitHub Actions to automate database schema changes and corresponding application deployments on ECS Fargate. Laridae standardizes a workflow that ensures the database and application code stay in sync and function normally as schema changes are executed safely, i.e., reversible.

Laridae creates infrastructure for hosting the core Laridae migration tool inside your own AWS VPC network and fully automates the process of making a schema change to your private PostgreSQL database and redeploying the application instance configured to utilize the updated database schema on ECS Fargate.

1.1) Design Goals

Laridae is a tool to automate database schema migrations and simplify the deployment of schema changes. We designed Laridae with the following goals in mind:

  • Convenience — The tool should be easy to install and configure.
  • Cost-Efficiency — The user should only incur expenses for the resources used on AWS, with no need for an opaque monthly fee.
  • Control — The tool should rely on the user's cloud infrastructure so the user can maintain control over their data.

1.2) Laridae Features

Laridae users can:

  • Initialize a GitHub Action workflow to:
    • Provision all AWS resources to create the infrastructure to host the core Laridae migration tool.
    • Execute the database schema changes in their database.
    • Upon completion of migration changes, redeploy the application instance configured to utilize updated schema changes on ECS Fargate.
    • Tear down all AWS resources used in said infrastructure.
  • In addition to automating the integration of database schema migrations, Laridaer:
    • It can be used as a standalone CLI tool, or the CLI tool can be integrated into your own pipeline.

1.3) What Laridae Supports

    Currently, Laridae only supports:
  • Applications using a PostgreSQL database.
  • Pipelines where the application code is deployed on ECS Fargate.
